Add quality tier system for model selection

Features:
- ADD 5 quality tiers with comedic escalation:
  - base: Mistral 7B Q4 (~4GB, 45 tok/s) - Daily use
  - max: Mistral 7B Q6 (~5.5GB, 40 tok/s) - "This is the max!" ‚Üê joke
  - nuclear: Qwen2.5 14B Q4 (~8GB, 25-30 tok/s) - Superior reasoning
  - ridiculous: Qwen2.5 32B Q4 (~18GB, 12-15 tok/s) - Near GPT-4
  - ludicrous: Qwen2.5 72B Q2 (~28GB, 5-8 tok/s) - Maximum quality

- ADD --quality flag to run command
  python terractl.py run --date 2027-7-14 --quality nuclear

- UPDATE .env templates with quality-specific model paths
  KOBOLDCPP_MODEL_BASE, KOBOLDCPP_MODEL_MAX, etc.
  KOBOLDCPP_QUALITY=base (default)

- UPDATE install command to only prompt for base model
  Other tiers are optional (commented placeholders)

Implementation:
- cmd_run() resolves quality tier to model path
- Fallback to KOBOLDCPP_QUALITY from .env if --quality not specified
- Console output shows selected quality + model filename
- Clear error if quality tier not configured

Changes:
- terractl.py: quality resolution logic in cmd_run
- .env.linux.dist, .env.win.dist: 5 model paths per tier
- README.md: quality tier examples
- install generates .env with base + commented optionals

Comedy: Users discover nuclear/ridiculous/ludicrous exist after "max"
